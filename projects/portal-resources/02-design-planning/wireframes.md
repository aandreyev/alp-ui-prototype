### Visual Design Notes
- **Headers**: H2 for main sections, H3 for subsections
- **Cards**: White background, subtle border, hover shadow
- **Icons**: 16px Lucide icons with muted foreground color
- **Hierarchy**: Indentation using padding-left, connecting lines optional. Do not use excessive indentation. Use light connecting lines to show nested structure
- **Actions**: Small outline buttons, primary for main actions
- **External Links**: Blue text with external link icon

## Interaction Flows

### Flow 1: Basic Resource Access
1. **User arrives at Matter detail** â†’ Sees Resources tab with badge count
2. **Clicks Resources tab** â†’ Loads hierarchical resource view
3. **Scans resource hierarchy** â†’ Identifies relevant section/resource
4. **Clicks resource card** â†’ Opens resource modal with details
5. **Reviews resource information** â†’ Confirms it's the right resource
6. **Clicks "Open Resource"** â†’ External resource opens in new tab
7. **Returns to ALP** â†’ Continues matter work with resource available

### Flow 2: VD Folder Exploration
1. **Accesses Resources tab** â†’ Sees VD Offering Folders section
2. **Clicks VD Folder link** â†’ SharePoint folder opens in new tab
3. **Browses precedent resources** â†’ Finds additional relevant materials
4. **Returns to ALP tab** â†’ Continues with matter-specific resources

### Flow 3: Filtered Resource Discovery
1. **Opens Resources tab** â†’ Sees all resources displayed
2. **Clicks filter dropdown** â†’ Selects "Forms" to narrow focus
3. **Reviews filtered results** â†’ Sees only form-type resources
4. **Uses search within forms** â†’ Types "assessment" to find specific form
5. **Clicks target resource** â†’ Opens modal and accesses form

### Flow 4: Error Recovery
1. **Clicks resource link** â†’ External system unavailable/error
2. **Sees error modal** â†’ Clear explanation of issue and options
3. **Copies link to clipboard** â†’ Can paste in browser or save for later
4. **Tries VD folder alternative** â†’ Accesses backup resource location

## Accessibility Considerations

### Keyboard Navigation
- **Tab Order**: Logical progression through interface elements
- **Focus Indicators**: Clear visual focus states for all interactive elements
- **Keyboard Shortcuts**: Space/Enter to activate, ESC to close modals
- **Screen Reader**: Proper ARIA labels and descriptions

### Visual Design
- **Color Contrast**: WCAG AA compliant contrast ratios
- **Focus States**: High contrast focus indicators
- **Text Size**: Scalable typography, readable at 200% zoom
- **Icon Labels**: Text alternatives for all icons

### Motor Accessibility
- **Click Targets**: Minimum 44px touch targets
- **Hover States**: Not required for functionality
- **Timing**: No auto-disappearing content
- **Motion**: Reduced motion preferences respected
